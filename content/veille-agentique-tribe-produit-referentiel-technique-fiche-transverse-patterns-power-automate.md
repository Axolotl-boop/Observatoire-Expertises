# Fiche transverse — Patterns & pièges Power Automate

> **Type** : référence transverse (pas un module)  ·  **Rôle** : les pièges récurrents, **payés une fois**, à re-vérifier sur tout flux construit ou cloné. Le SPECS les disperse dans ses logs de session (§9) ; cette fiche les consolide.
> **Fait foi** : SPECS §9 (provenance et contexte de chaque leçon).

## Lecture de fichiers (SharePoint)
- **`Include Nested Items = Yes`** (`viewScopeOption = RecursiveAll`) sur tout `Get files (properties only)`. Toute la matière vit en **sous-dossiers** → avec `No`, l'action ne lit que la racine et renvoie `value: []` **silencieux** (matière vide, l'agent répond son message d'accueil). **Suspect n°1** d'un assembleur qui « réussit » sans rien produire. **Un `Save As` réinitialise ce réglage** → revérifier après tout clonage.
- **Lire la SORTIE de l'action, pas que son statut.** Un flux peut être `Succeeded` avec N fichiers et 0 erreur en ayant injecté du vide (tell : l'agent accueille « colle-moi la matière »). Toujours **valider le contenu du livrable**.
- **Filtre : scoper par `{Path}`** (`contains(item()?['{Path}'], 'Veille-emploi')`, etc.) **en plus** du nom. La convention `AAAA-MM-JJ_source_hash` est partagée entre briques → sans le scope chemin, un `startsWith` mois attraperait aussi les autres briques.
- **Slug avec underscores des deux côtés** (`_<slug>_`) pour éviter les matchs partiels entre slugs proches.

## Écriture de fichiers / dossiers
- **`Create file` n'a pas de sélecteur de bibliothèque** → son `folderPath` se résout contre la **bibliothèque par défaut** du site (≠ celle ciblée par `Get files`). Symptôme : « Root folder is not found ». **Fix** : renseigner `folderPath` **via le sélecteur de dossier (📁)**, jamais à la main.
- **`Create new folder` + run-after-regardless.** Le dossier parent (mensuel `AAAA-MM/` ou trimestriel `AAAA-T#/`) n'est **pas auto-créé**. Insérer un `Create new folder` avant le `Create file`, et régler l'aval pour s'exécuter **même si la création échoue** (`Configure run after` = *is successful + has failed + is skipped*) → un dossier déjà existant n'empêche pas le dépôt. `Create new folder` a, lui, un **vrai sélecteur de bibliothèque** → son path est **relatif à la library** (pas de préfixe `/Documents partages`).
- **Falaise du 1er du mois** : tout flux déposant dans `/AAAA-MM/` casse au 1er passage d'un nouveau mois s'il n'a pas le pattern ci-dessus. À auditer sur **tous** les flux mensuels.
- **Cohérence dossier ↔ nom de fichier** : tirer le dossier et le nom de la **même source de date** (e-mail = `receivedDateTime` ; RSS = `publishDate`) — sinon décalage à la frontière de mois.

## Boucles
- **`For each` auto-imbriqué** : si on alimente une action avec un « Identifier » pris dans le **contenu dynamique d'une action qui renvoie un tableau**, Power Automate auto-emballe une 2ᵉ boucle → matière dupliquée, run figé. **Fix** : référencer l'item courant **par expression** `items('<Boucle>')?['{Identifier}']`, jamais le chip de l'action amont. **Un seul `Apply to each`.**
- **Fan-out multi-sortie** : `degree of parallelism = 1` sur la boucle externe **+** reset de la variable de matière **en tête d'itération**. Sans degree 1 → corruption de la variable partagée (bleed) ; sans reset → empilement inter-itérations.

## Fiabilité des agents (Copilot Studio)
- **`Do until` retry-jusqu'à-valide** autour de l'`Execute Agent` : gate = **présence de la section de clôture** (ex. `## Points de décision`, ou un sentinel `[[…]]`) → attrape **vide ET tronqué** d'un seul test. Limites `Count = 8` · `Timeout = PT30M`. Après la boucle, `Condition`/`Switch` (même test) : valide → dépôt propre ; sinon → fichier préfixé « INCOMPLET » + ping (l'échec **crie**, ne passe pas en silence). *(Le non-déterminisme d'orchestration des agents Copilot est un spectre vide ↔ tronqué ↔ complet, toute expertise, tout run — non réglable par instruction de prompt ; le correctif est **au niveau du flux**. GPT-4.1 n'en souffre pas → 0 retry observé, mais le filet reste.)*
- **Câblage à deux variables** : la variable **lue par l'agent** (matière, constante d'une retry à l'autre) doit être **séparée** de celle qui **porte le gate** (sortie agent, écrasée). Sinon l'itération 2 nourrit l'agent de **sa propre sortie ratée** → `Do until` pire qu'inutile. Bug **invisible au 1er run**, fatal à la 1re retry.
- **`Execute Agent and wait`** : le champ **`Message`** est caché sous **« Advanced parameters »** → le révéler et le câbler explicitement.
- **Agent en mode matière-injectée** : **aucune** connaissance `.md` (le grounding n'indexe pas le `.md`), **aucun** Outil/flux de fetch (un Outil désactivé → `FlowDisabled` → `lastResponse` vide), et un prompt **« lis la matière du message »** (pas « va la chercher »). Diagnostic d'un `lastResponse` vide, **dans l'ordre** : (1) test pane avec une **petite** entrée ; (2) erreur `FlowDisabled` → Outil parasite à détacher ; (3) sinon → c'est la **taille**.
- **Test pane Copilot Studio** : se **fige** sur les générations longues (~2 min) — ce n'est pas un plant. Tester via **`Execute Agent and wait`** (le chemin de la prod, sans la limite de la pane).

## AI Builder — prompts (variables d'entrée)
- **Recoller intégralement le corps d'un prompt écrase le jeton de variable dynamique.** Le texte se recolle, mais l'**encadré `matiere`** (variable d'entrée) **ne survit pas au collage** → il devient du **texte mort**, l'injection tombe. Symptôme : `promptTokens` retombe au **squelette du prompt seul** (~2 957 mesuré ici), la matière n'est plus injectée, le digest sort **vide / générique**. **Fix** : après **toute** réécriture du corps, **réinsérer l'encadré via le bouton `Input → Text`** — nom **exactement `matiere`** (c'est lui qui mappe sur `item/requestv2/matiere` que le flux remplit), **jamais au clavier**. Un nom divergent (`matière`, `Matiere`, `matiere1`) **casse le mapping** côté flux. Après insertion, vérifier dans le panneau des variables d'entrée qu'il n'y a **qu'une seule** `matiere` (un doublon auto-renommé `matiere1` est un signe que l'ancienne variable était encore là — garder celle nommée pile `matiere`, supprimer l'autre).
- **Marqueur du fix, plus fiable que de lire le digest** : `promptTokens` au run suivant. **45 000+** = injection rétablie ; **~2 957** = encadré non pris (variable orpheline, doublon, ou jeton pointant sur un nom légèrement différent). Le compteur tranche en binaire, sans consommer l'analyse du livrable.
- **Test à coût nul avant de relancer le flux** : le volet de test (à droite de l'éditeur) accepte un échantillon dans le champ `matiere` → digest complet = encadré bien posé ; sortie courte/vide = encadré non pris. Évite de brûler un run de flux entier pour vérifier.
- **Pendant exact** du piège « `Execute Agent` : champ `Message` à recâbler » → après réécriture, la variable doit être **re-liée, pas re-tapée**. La même classe d'erreur réapparaîtra partout où un éditeur distingue *texte* et *jeton de variable* : le texte se colle, le lien non.

## AI Builder — agent d'annotation / scoring en post-passe *(leçons v4.3)*
- **Un croisement, et la garde de survie d'un `[structurel]`, doivent exiger une donnée INTERNE (propriétaire).** Sinon une mention externe **non vérifiable** (« State of X », non groundé en V1) sert d'alibi et **blanchit un `[structurel]` poussé par un sponsor**. Externe × externe (deux newsletters, newsletter + State of X) = **convergence, pas croisement**. *Tell* : au test, l'agent gardait un `[structurel]` Atlassian en s'abritant derrière un « State of X » — corrigé en imposant le côté interne. Règle générale : **un critère de qualité qui peut être satisfait par une source que l'agent ne peut pas vérifier est un critère contournable.**
- **Ne jamais demander à un LLM de recompter sa propre sortie.** 3 tests : l'indice annonçait « 1 croisement · 7 non-corroborés » quand la liste en portait **0 · 10**. Compter est de l'**arithmétique** → ça se calcule en **déterministe** sur la sortie **structurée** (les `prov` du JSON), pas par le modèle. L'indice qualitatif (ce que le modèle nomme sûrement : où est l'ancrage, ce qui est externe-seul, N downgrades qu'il a lui-même posés) reste au LLM ; le décompte chiffré va au consommateur (dashboard/indexeur). *Même principe que le delta Boond et la passe captation.*
- **Énumérer positivement, jamais par exclusion négative — y compris dans une définition de comptage.** « non corroboré = ni propriétaire ni croisement » est un **compte par exclusion** : le modèle s'ancre sur le *label littéral* et rate les cas non labellisés. **Fix** : faire de `non corroboré` un **label positif** posé par signal → l'agent le pose, le décompte le tally. (C'est le même principe que pour les gates de seuil.)
- **Post-passe = couche ajoutée, pas réécriture.** Un agent de post-traitement qui doit *modifier* un texte long le **paraphrase** (drift). Le faire **n'émettre que sa couche** (annotations + bloc machine), et **concaténer** en aval `[texte original] + [couche]` → l'original reste byte-for-byte, zéro drift. La correction (downgrade) s'expose en annotation + dans le JSON, sans toucher le corps.
- **Filet de post-passe en autonome** : l'action d'annotation peut échouer (timeout). `Create file` en **run-after-regardless** + body `if(empty(coalesce(sortie,'')), texte brut, texte brut + couche)` → KO ⇒ on dépose le **brut** (publiable), jamais de perte. ⚠️ **`coalesce` obligatoire dans la branche concat** : PA évalue les **deux** branches du `if` avant de choisir → un concat sur sortie nulle planterait le cas même qu'on rattrape.


- **`month()` n'existe pas** → numéro de mois via `int(formatDateTime(X, 'MM'))`.
- **Piège « mode-mixte » du `@{}`** : `@{}` autour de **toute** l'expression (`@{formatDateTime(...)}`) ou en interpolation littérale (`fichier-@{expr}.json`) = **sain** ; `@{}` autour d'un **argument interne** d'une fonction qui doit, elle, s'évaluer (`formatDateTime(@{token}, 'yyyy-MM')`) = **cassé** (seul le token est résolu, le reste reste du texte littéral). **Fix** : ressaisir en **expression pure via l'éditeur `fx`**, sans `@{}`. **Tell visuel** : un **seul chip** = sain ; un nom de fonction en **texte gris** autour d'un chip = cassé.
- **Trailing slash (WordPress)** : `last(split(url, '/'))` rend **vide** si l'URL finit par `/` → slug vide → collision de noms. Retirer le `/` final avant de splitter.

## E-mail (captation)
- **Le forward réécrit l'expéditeur** (devient toi) **et préfixe le sujet `TR:`** → casse le mapping from/subject → slug. **Pour tester, déplacer (glisser-déposer) le mail original**, jamais le transférer.
